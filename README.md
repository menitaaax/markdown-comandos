# Documentación y comandos - Markdown 


## 1-Encabezados
Los encabezados se hacen usando el símbolo # al inicio de la línea.
Entre más # pongas, más pequeño es el título. Por ejemplo:

# Título grande
## Subtítulo
### Título pequeño


## 2-Negritas y cursivas

Negritas

Para negritas, debes poner dos asteriscos (*) al inicio y dos al final:
**Texto en negritas**

Para cursiva se usa un asterisco (*) al inicio y uno al final:
*Texto en cursiva*

Para ambos se usan tres asteriscos (*) al inicio y al final:
***Texto en negritas y cursiva***

## 3-Listas

Lista con puntos

Debes usar un guion (-) y un espacio:
- Manzana
- Sandía
- Uva

Lista numerada

Solo escribes el número + tetxo:
1. Paso uno
2. Paso dos
3. Paso tres

## 4-Enlaces
Para poner un enlace se usa:

[texto] para lo que va a aparecer

(link) para la URL

Por ejemplo: [Visitar GitHub](https://github.com/)

## 5-Imágenes
Es igual que un enlace, pero con un signo de exclamación (!) al inicio.

Por ejemplo: ![Atardecer en la playa](https://images.unsplash.com/photo-1500530855697-b586d89ba3ee?auto=format&fit=crop&w=200&q=40)

## 6-Citas
Para hacer una cita se pone > al inicio de la línea:

> Esto es una cita.

## 7-Código
Código corto (inline)

Para poner código dentro de una línea, se usa una comilla invertida (`) al inicio y al final:

`console.log("Hola");`

Bloque de código

Para poner un bloque completo debes poner tres comillas invertidas arriba y abajo:

```
function saludo() {
    return "Hola";
}
```


Divisor

Para crear una línea que divida secciones, se ponen tres guiones (---):

---

Tabla

Las tablas usan | para separar columnas y - para crear la línea de separación:

| Comando      | Función        |
|--------------|----------------|
| **Negritas** | Resaltar texto |
| *Cursiva*    | Énfasis        |

# Más ejemplos de Markdown

Aquí agregamos unos comandos extra que estuve probando y que también sirven mucho cuando estás haciendo documentación.

**-Texto tachado**

Para tachar texto se usan dos virgulillas (~~) al inicio y al final:

~~Este texto está tachado~~

**-Lista de tareas (checklist)**

Sirve para hacer tareas pendientes. Se usa - [ ] para sin marcar y - [x] para marcada:

- [ ] Falta hacer este punto
- [x] Este ya está completo

**-Línea de cita doble**

Puedes citar dentro de otra cita usando >>:

> Primera cita
>> Cita dentro de otra
